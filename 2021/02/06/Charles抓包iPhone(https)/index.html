Charles抓包iPhone(https)

老杜
2021-02-06
手机
  • 配置
  • 很重要的一点
  • 手机网络设置
  • 原理

配置

  • Proxy->SSL Proxying Settings:

    • Enable SSL Proxying (√)
    • location->add
      • host [*]
      • port [443]

  • Proxy->Access Control Settings:

    • 0.0.0.0/0 (all)

  • help->SSL Proxying:

    • Install Charles Root Certificate
      • 钥匙串(Keychain)找到对应的 Charles Proxy CA
      • 打开->信任->始终信任

  • help->Install Charles Root Certificate on a Mobile Device or Remote Browser

    • 按照提示下载+安装证书

很重要的一点

  • iOS10 以上系统请到设置->通用->关于->证书信任中找到对应证书(Charles)打开信任开关

手机网络设置

  • Wlan->找到电脑相同的网络->(i)->代理设置:
    • ip:电脑 ip 地址
    • port:Charles->Proxy->Proxy->Settings 对应端口
  • 至此手机的请求就可以被 Charles 截获了

原理

  • 一般的HTTPS请求分为以下几步:
  • 1、客户端访问HTTPS连接, 并把安全协议版本号、客户端支持的加密算法列表、随机数C发给服务端;
  • 2、服务端返回非对称加密的公钥SCER、商定的加密算法、随机数S
    • 服务端接收密钥算法配件后, 会和自己支持的加密算法列表进行比对, 选择一种对称算法(如AES)、一种公钥算法(如具有特定秘钥长度的RSA)和一种MAC算法发给客户端; 服务器端有一个密钥对, 即公钥和私钥, 是用来进行非对称加密使用的, 服务器端保存着私钥, 不能将其泄露, 公钥可以发送给任何人;
  • 3、客户端会对server公钥进行检查, 验证其合法性, 如果发现发现公钥有问题, 那么 HTTPS传输就无法继续;
  • 4、如果公钥合格, 那么客户端会用服务器公钥来生成一个前主秘钥(Pre-Master Secret, PMS), 并通过该前主秘钥和随机数C、S来组装成会话秘钥;
  • 5、客户端通过服务端的公钥来对前主秘钥进行非对称加密, 发送给服务端;
  • 6、服务端通过私钥解密得到前主秘钥;
  • 7、服务端通过前主秘钥和随机数C、S来组装会话秘钥;
    • 至此, 服务端和客户端都已经知道了用于此次会话的主秘钥;
  • 数据传输
    • 客户端收到服务器发送来的密文, 用客户端密钥对其进行对称解密, 得到服务器发送的数据;
    • 同理, 服务端收到客户端发送来的密文, 用服务端密钥对其进行对称解密, 得到客户端发送的数据;
  • Charles介入的请求核心作弊行为是对客户端的欺骗:
  • 1、客户端访问HTTPS连接, 并把安全协议版本号、客户端支持的加密算法列表、随机数C发给服务端(其实是发送给了Charles);
  • 2、当客户端和服务器建立连接时, Charles会拦截到服务器返回的证书(服务器公钥)和随机数S;
  • 3、然后动态生成一张伪造证书（Charles公钥【假公钥】)发送给客户端;
  • 4、客户端收到Charles伪造证书后, 进行验证; 因为之前我们手机设置了信任, 所以验证通过;
  • 5、客户端用伪造的Charles证书生成PMS并用其加密发送给服务器(其实是发送给了Charles);
  • 6、Charles通过伪造的Charles证书的私钥解密, 就得到了客户端的 PMS;通过PMS+C+S就可以得到跟客户端对称的会话密钥;
    • 至此, 客户端的“密文”请求就以“明文”的形式展现在Charles上了;
  • 7、在第1步中Charles已经获得了客户端的随机数C; 在第2步中获得了服务端证书和随机数S; 那么就可以做出PMS+C+S的【真会话密钥】, 并以客户端的身份把通过服务器公钥加密的PMS发给服务器, 就得到了跟服务器对称的会话密钥;
    • 至此, 服务器的“密文”请求就以“明文”的形式展现在Charles上了;
  • 数据传输
    • Charles在客户端和服务器之间, 手握两端对称的会话密钥;
    • 对客户端谎称自己是服务器;
    • 对服务器谎称自己是客户端;
    • 所有的数据都会在Charles里过滤一遍, 所谓的"密文"已经以"明文"的形式暴露在抓抱者面前了;
