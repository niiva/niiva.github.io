小结

  • 下面来解释一下 Dockerfile 里面的各种常用参数:

    参数 说明
    FROM 基础镜像, 当前新镜像是基于哪个镜像的
    RUN 容器构建时需要运行的命令【需要注意的是每一行 RUN 指令都会在 docker 上新建一层, 一般使用&&符号把多行命令连接起来, 保证只有一个 RUN 指令】
    EXPOSE 当前容器对外暴露出的端口【EXPOSE 指令是声明运行时容器提供服务端口, 这只是一个声明, 在运行时并不会因为这个声明应用就会开启这个端口的服务; 在 Dockerfile 中写入这样的声明有两个好处, 一个是帮助镜像使用者理解这个镜像服务的守护端口, 以方便配置映射; 另一个用处则是在运行时使用随机端口映射时, 也就是 docker run -P 时, 会自动随机映射 EXPOSE 的端口;】
    WORKDIR WORKDIR 指令为 Dockerfile 中的 RUN指令设置工作目录【举个例子, 比如上面的webpy安装包里面有../xxx的写法, 如果不指定工作目录, 则会提示找不到xxx文件夹的错误】
    ADD 对目标镜像进行文件/目录的添加, ADD 命令会自动处理 URL 和解压 tar 压缩包
    COPY 类似 ADD, 拷贝文件和目录到镜像中, 但是不能添添加网络文件(URL)
    CMD 知道一个容器启动需要运行的命令, Dockerfile 中可以有多个 CMD 指令, 但只有最后一个生效, CMD 会被 docker run 之后的参数替换
